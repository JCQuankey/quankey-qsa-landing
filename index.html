How It Works

Three commands cover the entire workflow: scan, compare, and verify.

# Scan endpoints from a target list
qsa scan --targets endpoints.txt --output ./results

# Compare two scans for drift over time
qsa diff baseline.json current.json

# Verify evidence pack hash-chain integrity
qsa verify evidence-pack.json

# Run from Docker (no installation needed)
docker run --rm qsa scan host.example.com:443

Output includes a self-contained HTML report with endpoint inventory, findings, scores, and classification rationale. The JSON evidence pack preserves full hash-chain integrity for audit trails.
